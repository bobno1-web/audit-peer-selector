# skill: measurement_definitions — "성공률"의 정의를 고정한다

## 언제
추출률/성공률/coverage 같은 비율을 보고할 때마다.

## 무엇을
- **"성공률"은 항상 "필요 항목 전부 확보" 기준**이다.
  (예: 필수 6계정이면, 6개 **전부** 나온 케이스만 성공.)
- **"일부라도 나오면 성공" 금지.** 관대한 정의로 헤드라인을 부풀리지 않는다.
- **헤드라인과 세부가 다른 얘기를 하면 헤드라인을 버린다.**
  (계정별로는 다 70%가 넘어도, "6개 전부"가 51%면 기준 지표는 51%다.)
- 애매하면 낮은 쪽(엄격한 정의)을 채택한다. 기준을 늘려잡지 않는다.

## 근거 (실제 사고)
- **0-D**: "경로3 100%"라고 보고했으나 실제로는 "최소 1개 계정 확보"였다.
  전부-확보 기준으로 다시 재면 훨씬 낮았다.
- **0-E**: 계정별은 전부 ≥70%였으나 6계정 **전부** 확보는 51% → NO-GO(D-008).
  느슨히 봤으면 GO로 오판할 뻔했다.
