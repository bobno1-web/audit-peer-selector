# skill: spec_derivation — "필요한 것 목록"을 사람이 타이핑하지 않는다

## 언제
"필수 계정", "채점 대상 비율", "허용 입력" 같은 **목록**을 다룰 때마다.

## 무엇을
- 목록을 **손으로 타이핑하지 않는다.**
- 원천 문서(`docs/ORACLE.md`, `config/`)에서 **스크립트로 유도**한다.
  (예: `scripts/derive_required_accounts.py` 가 ORACLE 비율 → config 매핑 → 필수 계정 집합.)
- 유도 산출물 파일에는 "이 파일은 스크립트 산출물이다. 손편집 금지"를 명시한다.
- 사람이 타이핑한 목록은 원천과 어긋나도 아무도 모른다. 유도하면 어긋날 수 없다.

## 근거 (실제 사고)
- **0-D**: 필수 계정 목록에 **종업원수**를 손으로 넣었다. 그런데 종업원수는
  ORACLE의 어떤 채점 비율에도 안 쓰인다. 이 오류 하나로 비상장 유니버스(11만+)를
  통째로 폐기할 뻔했다(0-E에서 D-007로 철회·기계 유도로 교체).
