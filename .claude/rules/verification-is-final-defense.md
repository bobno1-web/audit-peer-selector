# 규칙: 검증방이 최후 방어선 (verification-is-final-defense)

## 무엇을 인정하는가 (무한후퇴)
게이트 판정을 코드로 강제해도, **그 강제 코드(`gate.py`, `gate_metrics` 재집계 등)도 에이전트가
수정할 수 있다.** 방어 코드를 지키는 또 다른 코드를 두어도 그것 역시 수정 가능하다 → **무한후퇴.**
어떤 자기방어 코드도 자신을 수정하는 주체로부터 자신을 완벽히 지킬 수 없다.

## 그래서 어디에 정박하는가
**최종 안전장치는 검증방(사람-분리 검증)의 독립 재집계다.**
- 검증방은 **개발방 코드를 신뢰하지 않는다.** `runs/` 원자료에서 **직접** 재계산해 대조한다.
- 게이트 통과·점수 주장은 **검증방의 독립 재현 없이는 확정되지 않는다.**
- 개발방의 `gate.py judge`·`gate_metrics` 는 **1차 방어**(합리화·우발적 위조 차단)일 뿐,
  최종 확정이 아니다. 최종 확정은 검증방 몫이다.

## 실무
- 개발방: 원자료를 `runs/` 에 남기고(재집계 가능하게), 판정 경로를 코드로 두되 **그것이 최종이라
  주장하지 않는다.**
- 검증방: `runs/` 원자료로 독립 재집계 → 개발방 수치와 대조 → 일치해야 확정.
- 불일치·원자료 부재 → 미확정(FAIL 취급).

## 근거 (실제 발견)
- Loop 0-I 검증: judge 재집계를 코드로 넣었으나, **그 재집계 코드도 수정 가능**이라 무한후퇴
  (검증 #1,#2,#4). → 코드 방어를 최종으로 착각하지 말고, **인간-분리 검증에 정박**한다.
  `DECISIONS` D-019.
