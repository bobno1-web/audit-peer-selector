# 규칙: API 비용 가드 (api-budget)

## 무엇이 금지인가
외부 API·임베딩을 **필요 이상으로** 호출하는 것. 특히:
- 같은 텍스트를 두 번 임베딩/벡터화하는 것.
- 사업보고서 **전문**을 임베딩하는 것 (필요한 섹션만).
- 재실행 때 캐시를 확인하지 않고 처음부터 다시 호출하는 것.
- 대량 호출 전에 **예상 호출 수를 로그로 남기지 않는** 것.

## 왜 위험한가
외부 API(OpenDART 문서 취득, KRX 시세, 임베딩 API)는 **유료·한도(rate/daily limit)**가 있다.
캐시 없이 반복 호출하면 비용이 폭발하고 한도(020)에 걸려 루프가 멈춘다.
전문 임베딩은 신호(사업모델) 대비 노이즈(재무·주석·법률)만 키운다 — Loop 2에서 실증됨(텍스트 가중치 0).

## 규칙 (코드로 강제)
1. **캐시 필수.** 모든 외부 취득물(문서 섹션 텍스트, 시총, 벡터)은 디스크에 캐시한다.
   재실행 시 캐시를 **먼저** 확인하고 **캐시 미스만** 호출한다.
2. **캐시 키 = 콘텐츠·구성의 해시.** (기업, 연도/시점, 섹션소스, 모델/벡터라이저 버전)을
   해시해 키로 쓴다. 소스·버전이 바뀌면 키가 바뀌어 **새 캐시로 분리**된다(구캐시 오염 없음).
3. **섹션만.** 사업보고서 전문이 아니라 필요한 섹션("사업의 내용")만 처리한다.
4. **예상 호출 수 로그.** 대량 호출 전 `to_fetch=N (cache_hit=H, cache_miss=M)`를 stderr에 남긴다.
5. **히트/미스 보고.** 실행 종료 시 캐시 히트/미스 수를 출력에 보고한다.

## 기계적 강제
- `tests/test_embedding_cache.py`: 같은 키를 두 번 요청하면 두 번째는 취득/벡터화 호출이 **0**임을
  카운터로 실증한다. 캐시 파일이 있으면 재실행 시 호출 0임을 실증한다.
- 벡터 캐시는 `(콘텐츠해시, 벡터라이저버전)`를 npz 메타에 기록 — 불일치 시에만 재계산.

## 근거
- **Loop 2**: 사업보고서 전문(첫 4000자, 사실상 표지+목차)을 flatten TF-IDF로 써서
  텍스트 가중치가 0이 됐다(신호가 노이즈에 묻힘). → Loop 3은 섹션만 정밀 추출하고,
  재취득·재벡터화 비용을 캐시로 막는다.
