# FINAL_REPORT — 동종업계 탐색기 최종 성능 검증 (Loop 7, holdout 개봉)

## 프로젝트 목표
기업 하나를 넣으면 **비교가능한 동종기업(peer)을 순위대로** 뽑는 엔진. 용도 = 초도감사(신규 수임)의
**분석적 절차**: 타겟의 재무비율이 동종기업 대비 이상한지로 감사 위험을 탐지한다. 좋은 peer 를 뽑으면
"정상 범위"가 정확해진다. **채점** = peer 4개 재무비율(매출총이익률·영업이익률·재고자산회전율·매출채권회전율)의
**중앙값을 타겟 예측치**로 삼아 실제와의 오차(APE) 중앙값. 낮을수록 좋다.

★ **정보 차단벽**: 엔진은 채점 대상 비율을 계산할 수 있는 계정을 볼 수 없다(산업·규모·시총·사업내용텍스트·
성장률만). 실제 감사인이 그 숫자를 아직 모르는 상황을 재현.

## ★★ 최종 판정 3종

### (a) similarity(L6)가 baseline 을 out-of-sample 에서 이기는가? — **예, 유의하게.**
- holdout: **L6 0.5029 vs baseline 0.5585 = −10.0%.** firm-clustered 부트스트랩 CI **[−0.0737, −0.0370]**
  → 0 미포함, **유의**. paired 도 [−0.0642,−0.0465] 유의.
- L6 는 **k=10 개선도 holdout 에서 유지**: vs L4 −0.0269, clustered CI [−0.0423, −0.0102] **유의.**
  → Loop 6 의 "peer 5→10 이 도움"이 dev 과적합이 아니라 **out-of-sample 실재**임을 확정.

### (b) 과적합이었나, 진짜였나? — **진짜다(과적합 경미).**
- ★ **dev 순위가 holdout 에서 완전히 유지**: [L6 < L4 < L3 < L2 < baseline] 순서 그대로. 텍스트·시총·성장·
  k=10 각 개선이 **out-of-sample 에서도 순위대로 성립** → 학습이 dev 에 과적합한 게 아니라 일반화됨.
- 저하율 +2.9~6.4%(경미). similarity 축(L2~L4)이 baseline 보다 약간 더 저하(학습 가중치의 소폭 과적합)
  하지만, **L6 의 k=10 이 오히려 안정화**(L6 +4.9% < L4 +6.4%) — 더 많은 peer 가 out-of-sample 견고성↑.
- L6 4비율 **전부** holdout 에서도 L4보다 낮음(영업이익률 0.721<0.763, 매출채권 0.393<0.413,
  재고자산 0.576<0.598, 매출총이익 0.392<0.425).

### (c) 승리조건(0.433) 최종 충족? — **미달(전체·안정 모두).**
- **holdout 전체 0.5029 > 0.433**, **holdout 안정 0.4657 > 0.433.** 둘 다 미달. dev 에서도 미달(0.4794)이었고
  out-of-sample 에서 **확정**. 승리조건(baseline −20%)은 이 방식의 사거리 밖.

## ★ 정직한 최종 결론

**이 도구는 실사용 가능한가? — 조건부 예.**
- **검증된 것:** peer 선정이 **소박한 산업+규모 매칭(baseline)보다 out-of-sample 로 유의하게(−10%) 낫다.**
  개선들(텍스트·시총·성장·peer수)이 전부 실재(순위 유지). 즉 "이 엔진이 뽑은 peer 로 만든 정상범위"가
  baseline 보다 정확하다 — 감사 분석적 절차의 **상대적 개선 도구**로 쓸 수 있다.
- **못하는 것:** 재무비율을 목표 정밀도(APE 중앙값 0.433)로 **예측하지 못한다.** out-of-sample 상한은
  **≈0.50(median APE)**. 즉 "peer 중앙값으로 타겟 비율을 맞히기"의 남은 오차는 **동종그룹 내 사업
  이질성**이라 peer 선정 정교화로는 더 못 줄인다(Loop 2~6 의 무료·유료 표현축이 전부 이 상한에 수렴).

**"peer 선정으로 재무비율 예측"의 실증된 상한:**
- **median APE ≈ 0.48(dev) / 0.50(holdout).** baseline 대비 −10% 개선이 한계. 이 상한은 6개 축(산업·규모·
  시총·텍스트·성장·부문)·예측방식(k·집계함수)·신경망 임베딩을 **전부 실측**해 도달한 것이다(구현/데이터
  부실 아님 — 정보가 그만큼밖에 없다).

**이 방식이 못 다루는 범위:**
- **실적 0 근처(손익분기) 기업**: 비율 분자가 0 근처면 APE 가 구조적으로 폭발(ill-conditioned). dev 유도
  기준으로 **9.9%**가 이 범주 — 이들은 "동종업계 비교 자체가 부적합"(안정 APE 로 분리 보고).
- **회계정책 차이**(수익인식·재고평가): 사업 차이와 구별 못 함(ORACLE 한계, 감사인 판단 몫).
- **비상장·초대형/초소형**: 유니버스가 상장사라 peer 규모 편향(D-008 구조적 한계).

## ★ 산출물 사용 가이드 (감사 실무)
1. **peer 목록 사용:** 타겟을 넣으면 L6 가 상위 peer(기본 10개)를 순위로 낸다. 그 peer 들의 4비율
   **중앙값·분위수 범위**를 "정상 범위"로 삼아 타겟 비율의 이탈을 본다. 이탈이 크면 감사 **red-flag**.
2. **신뢰 범위(어디서 믿고 어디서 조심):**
   - **믿을 만:** 정상 영업(영업이익이 매출 대비 유의미), 상장 중견, 산업분류 명확한 기업. 이런 곳에서
     L6 정상범위는 baseline 보다 −10% 정확.
   - **조심:** ① **손익분기 근처(영업이익/총자산 하위 10%)** → "비교 부적합" 표시, 비율 이탈을 red-flag 로
     읽지 말 것. ② peer 유효수 <3 → 저신뢰(FAIL 페널티 대상). ③ 시총·성장 결측 기업 → 유사도 축 일부 미작동.
   - within 정확도(holdout): 타겟 비율의 **10%/20%/30% 이내** 예측 비율 = 10.8% / 21.2% / 31.5%.
     즉 "peer 중앙값이 실제의 ±20% 이내"가 약 1/5 — **점추정보다 범위·이상탐지 용도**로 쓰는 게 맞다.
3. **기존 비교 툴 결합:** 이 엔진의 peer 목록을 기존 산업분류 기반 비교 리스트의 **대안·보완**으로.
   단 절대 예측치가 아니라 **상대 순위·정상범위 산출기**로 위치시킨다.

## ★ 다음 단계 권고 (정교화 — 예측오차 저하 아님, 실무 활용도 향상)
숫자(APE)는 상한에 도달했으므로, 남은 여지는 **예측 정확도가 아니라 산출물 표현**이다:
1. **신뢰도 등급:** 각 (타겟, 비율) 결과에 신뢰 등급 부착(peer 유효수·유사도 분산·ill-conditioned 여부 기반).
2. **선정 근거 설명:** 왜 이 peer 가 뽑혔는지(산업 일치·규모 근접·사업내용 유사) 축별 기여를 노출.
3. **red-flag / 비교부적합 표시:** 이탈 red-flag 와 "구조적 비교 부적합"(손익분기 근처)을 **구분 표기**.
4. **출력 형식 확정:** peer 목록 + 정상범위(분위수) + 타겟 위치 + 신뢰등급의 표준 리포트 포맷.
- ★ **이것들은 holdout 재사용이 아니다.** 예측 오차를 낮추려는 재튜닝이 아니라, **이미 확정된 성능을 실무에
  쓰기 좋게 표현**하는 개선이다. holdout 은 이 검증으로 **소진**됐고, 재예측 튜닝에 다시 쓰지 않는다.
