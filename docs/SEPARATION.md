# SEPARATION — 예측 불가능(ill-conditioned) 기업 분리 (Loop 6 PART 2, C)

## 진단 — 왜 일부 기업은 어떤 방식으로도 예측이 안 되나
APE = |예측 − 실제| / |실제| 는 **|실제| → 0** 에서 ill-conditioned(작은 절대오차가 비율로 폭발).
비율 실제값 = |분자/분모| 이 0 근처가 되는 구조적 원인 = **그 비율의 분자 계정이 스케일 대비 0 근처**
(영업이익·매출총이익 ≈ 0 = 손익분기 근처 기업). 큰 오차의 다수가 이 구조에서 나온다. 이것은
**우리 예측 실패가 아니라 타겟 자체의 재무구조**이며, 어떤 peer 방식으로도 안정적으로 못 맞춘다.

## 판정 기준 (구조적·dev 유도, D-027) — ★ 정답 훔쳐보기 아님
- 각 (기업×시점×비율) 케이스에서 **s = |비율_분자계정| / 총자산**.
- **s < τ_r 이면 '구조적 예측불가' 플래그.** τ_r = 그 비율의 **dev 분포 하위 분위수**(q_sep=0.10,
  config; penalty 의 상위꼬리 q=0.9 와 대칭인 하위꼬리). 값은 dev 에서 유도(임의상수 아님).
- ★ **타겟의 채점비율 실제값을 쓰지 않는다**(그건 정답 훔쳐보기). 원천 재무구조(분자 계정·총자산)만.
  총자산은 엔진 허용 입력(스케일). 비율별 예외분기 없음(4비율 공통 s=|분자|/총자산).
- ★ **왜 분모가 아니라 분자 near-zero 인가:** 회전율(매출원가/재고자산)은 분모(재고)가 0 근처면
  actual 이 **커져** APE 가 ~1 로 유계(폭발 아님). 폭발은 오직 **actual→0 = 분자→0**. 지시서 예시
  (영업이익 0 근처)와 일치.
- 총자산 결측 → s 계산 불가 → **플래그 안 함**(전체·안정 모두 잔류).

## 결과 — 전체 vs 안정 (둘 다 보고, 몰래 빼기 아님)

- **제외 비율 = 9.9%**(< 20%). q_sep=0.10 을 비율별로 적용한 결과. τ_r 은 각 비율 dev 하위 10% 지점.
- ★ **플래그는 타겟 재무구조 기반이라 방식(시장·baseline·L4·L6) 무관하게 동일 집합** → 모든 대조군에
  **같은 split** 적용 → 상대 비교 불변(L6 만 유리하게 빼는 것 아님).

| 엔진 | 전체 APE 중앙값 | 안정 APE 중앙값 | Δ(안정−전체) |
|---|--:|--:|--:|
| 시장중앙값(A) | 0.5724 | 0.5378 | −0.035 |
| baseline | 0.5427 | 0.5016 | −0.041 |
| L4 (median@k5) | 0.4977 | 0.4627 | −0.035 |
| **L6 (median@k10)** | **0.4794** | **0.4452** | −0.034 |

(교정 targets. 전체 수치·부트스트랩은 SHOWDOWN_L6.)

- **안정 APE 가 전체보다 약 0.035~0.04 낮다** — ill-conditioned 9.9% 를 제외하면 나머지에서 **더 나은
  성능이 드러난다.** 주지표가 중앙값(꼬리에 견고)임에도 0.035 이동한 것은, 손익분기 근처 기업의 높은
  APE 가 상위 꼬리뿐 아니라 **중앙 부근까지 밀어올려** 있었음을 뜻한다.
- **상대 비교는 불변:** 안정 기준에서도 L6 의 시장 대비 우위가 유지(오히려 소폭 확대).

## 감사 활용 — "비교 부적합" 플래그 (red-flag 아님)
이 플래그의 산출물 가치: **"이 회사는 이 비율로 동종업계 비교가 통하지 않는다"**는 정보다.
- 손익분기 근처(영업이익≈0) 기업은 마진 비율이 구조적으로 불안정 → 감사인이 그 비율의 peer 대비
  이상치를 볼 때 **"방식 한계로 신뢰도 낮음"**을 함께 표시한다.
- ★ 이것은 **red-flag(이상 신호)가 아니다.** red-flag 는 "정상 범위에서 튄다"이고, 이 플래그는
  "애초에 정상 범위를 정의하기 어렵다(비교 부적합)"이다. 둘을 구분해 표시해야 감사인이 오해하지 않는다.
- 산출물에서: 각 (기업, 비율) 결과에 `comparable=False` 플래그를 부착해, peer 대비 편차를 보고할 때
  ill-conditioned 케이스는 **별도 구획**으로 분리(점수에서 몰래 빼는 게 아니라 사용자에게 명시).

## 한계 (정직히)
- 제외 9.9% 는 임계선 미만이나 작지 않다 — dev 케이스의 ~1/10 이 이 방식으로 **구조적 예측 부적합**.
  이는 "peer-중앙값 방식"의 적용 범위 자체의 한계를 정량화한 것(감사 관점에서도 유용한 정보).
- s=|분자|/총자산 은 분자 near-zero 의 **프록시**다(고마진·저자산회전 기업이 오플래그될 수 있음).
  그러나 구조적·사전등록·균일 적용이라 특정 기업 체리피킹이 아니다. 분모(재고·매출채권) near-zero 로
  actual 이 커지는 유계 케이스는 의도적으로 플래그하지 않는다(폭발 아님).
