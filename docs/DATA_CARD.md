# DATA_CARD — Point-in-Time 데이터 (Loop 0-H 전량, 게이트 PASS)

> ✅ **검증 완료.** 생존편향 게이트 PASS(코드 판정, `DECISIONS` D-016) 후 **유니버스 전량**을
> 적재했다. (0-F의 220-표본·0-G "미검증" 상태는 대체됨.) 리더 `pit.as_of(date)`, 원천 OpenDART.
> 숫자는 실측(빌드 산출 parquet 재집계, `scripts/pit_build_full.py`).

## 구조
```
data/pit/
├── features/   ← 엔진이 읽음(허용 입력)
│   ├── scale/      scale_<Y>.parquet    매출액, 총자산
│   ├── industry/   industry_<Y>.parquet 산업분류(induty_code)
│   └── business/   (Loop 2 지연 — README)
├── targets/    ← 채점기 전용(엔진 접근 금지)
│   └── ratios/     ratios_<Y>.parquet   매출액·매출원가·매출총이익·영업이익·재고자산·매출채권
└── universe/   universe_<Y>.csv         시점별 유니버스(생존편향-안전)
```
- **접근은 오직 `pit.as_of(date)`.** rcept_dt(제출일) ≤ date 만 반환(미래 차단).
- **정보 차단벽**: `as_of(T).features` 에 채점 계정 컬럼 없음(테스트 강제).

## 규모 (기업 수 × 시점 수 = 케이스 수) — 전량
- **시점**: 10개 (2016~2025, 매년 5/15). FY2014 재무 부재로 2015 제외(아래 한계).
- **케이스 = 유니버스 전량 × 시점 = 24,767 corp-years** (표본 아님).
  - **dev(2016~2022): 16,465** / **holdout(2023~2025): 8,302** (holdout 봉인).
- 시점별 유니버스: 2121(2016) → 2854(2025) (생존편향-안전, `SURVIVORSHIP.md`).

## ★ 비율별 정의가능률 (전량 재집계) — ORACLE "타겟별 정의된 비율만 채점"
어떤 비율이 채점되는지는 그 기업의 **재무구조**가 정한다(분자·분모 존재 + 분모>0). 엔진 무관.
| 비율 | 정의가능률 | | 비율 | 정의가능률 |
|---|--:|---|---|--:|
| 영업이익률 | 71.5% | | 매출총이익률 | 65.6% |
| 매출채권회전율 | 70.6% | | 재고자산회전율 | **63.2%** |

- **재고자산회전율 63.2%**: 서비스·IT·금융은 재고자산이 구조적으로 없어 이 비율이 **정의되지 않음**
  (결측 아님 — 그 회사에 그 개념이 없음). 그 기업은 나머지 정의된 비율로만 채점된다.

## ★ 타겟당 정의된 비율 수 분포 (보조지표, 전량)
| 정의된 비율 수 | 4개 | 3개 | 2개 | 1개 | **0개** |
|---|--:|--:|--:|--:|--:|
| 비율(%) | 59.4 | 7.8 | 4.1 | 1.8 | **27.0** |
- **0개 = 27.0%**: 재무제표 자체가 없는 기업(`fnlttSinglAcntAll` 013 — 비표준 XBRL·일부 KONEX·
  지주 등, 또는 T 이후 정정본). ORACLE 명세대로 **벤치마크에서 제외**(임의 채움 없음).
- 산업분류: **100%** 커버(current-state, 아래 한계 3). 규모(매출액·총자산)는 재무 확보 기업에서 제공.

## 결측 처리
- **임의값 대체 없음.** 결측은 parquet `<NA>`. 0/평균 채움 금지. 정의 안 되는 비율은 그 기업에서 제외.

## 알려진 한계 (정직히)
1. **FY2014 재무 미제공**: `fnlttSinglAcntAll` 은 FY2015부터. → **시점 범위 2016~2025**(유니버스
   레이어는 2015도 존재하나 재무 없음).
2. **정정본 보수 처리**: 표준화 재무의 rcept_dt 가 T 이후면 그 시점 미가용으로 보고 **null**
   (원본 as-of-T 수치를 API로 못 얻음). 룩어헤드 대신 결측.
3. **산업분류 current-state**: `induty_code` 는 `company.json` 현재값(DART 개정 이력 미제공).
   100% 채워지나 point-in-time 아님. 산업은 거의 안 바뀌어 룩어헤드 위험 낮음(명시).
4. **사업내용 텍스트**: Loop 2 지연(구조만). 타당성 0-E 실측(96.7%).
5. **fs_div**: 개별(OFS) 우선·연결(CFS) 대체. `fs_div` 컬럼 기록.
6. **매핑 교정(Loop 6-B) 정정공시 미적용 2,291건**: 매출채권 별칭 버그(D-026) 교정을 위해 dev targets 를
   재생성할 때, `fnlttSinglAcntAll` 가 **T 이후 정정본**을 반환한 firm-year **2,291건**은 rcept_dt 불일치로
   **옛 값 유지**(정정본 사용=룩어헤드이므로 차단). 원본 filing 을 이 API 로 재취득 못 해 이들엔 교정
   **미적용**(옛 값 불변, 안전). 실제 교정 필요분은 767 보다 많을 수 있으나 **PIT-clean 하게 교정 가능한
   767 firm-year 만** 반영. 출처: `runs/2026-07-16_regen_targets/provenance.json`. (제안 [M2]: rcept_no 이력
   API 로 원본 filing 고정취득 확장 — 별도 루프.)
