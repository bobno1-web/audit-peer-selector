# DATA_CARD — Point-in-Time 데이터 (Loop 0-F PART C)

> ★★ **미검증 (Loop 0-G A-3).** 아래 220-기업 표본 PIT 는 **게이트 위반 상태에서 구축**됐다
> (0-F 게이트 FAIL을 넘어 진행 — `DECISIONS` D-011). **재무 데이터는 유니버스와 독립이라 폐기하지
> 않고 보존**하지만, **미검증**으로 표시한다. 유니버스 레이어와 전량 패널은 **0-G PART D 게이트
> 재판정이 PASS(사람 승인)된 뒤에만** 재생성·확정한다. 그 전까지 이 문서 수치는 참고용이다.

산출: `scripts/pit_build.py` (seed 20260714). 리더: `pit.as_of(date)`. 원천: OpenDART.
숫자는 실측(빌드 산출 parquet 재집계).

## 구조
```
data/pit/
├── features/   ← 엔진이 읽음(허용 입력)
│   ├── scale/      scale_<Y>.parquet    매출액, 총자산
│   ├── industry/   industry_<Y>.parquet 산업분류(induty_code)
│   └── business/   (Loop 2 지연 — README)
├── targets/    ← 채점기 전용(엔진 접근 금지)
│   └── ratios/     ratios_<Y>.parquet   매출액·매출원가·매출총이익·영업이익·재고자산·매출채권
└── universe/   universe_<Y>.csv         시점별 유니버스(PART B)
```
- **접근은 오직 `pit.as_of(date)`.** rcept_dt(제출일) ≤ date 만 반환(미래 차단, C-3).
- **정보 차단벽**: `as_of(T).features` 에는 채점 계정이 **컬럼으로도 없다**(테스트 강제).

## 규모 (기업 수 × 시점 수 = 케이스 수)
- **시점**: 11개 (2015~2025, 매년 5/15).
- **시점별 표본**: 220 기업 → **케이스 = 220 × 11 = 2,420 corp-years.**
  - dev(2015~2022): 1,760 / holdout(2023~2025): 660. (holdout 봉인.)
- ★ **표본 패널이다.** 전 유니버스(시점별 ~1,968~2,854, PART B)의 부분집합.
  전량 구축은 재무 호출 ~2.6만 건(일 API 한도 초과)이라 **Loop 1에서 예산 내 재실행**한다.
  (`pit_build.py` 의 `N_PER_YEAR` 만 키우면 됨. 기계·무결성은 표본으로 이미 검증.)

## 결측 처리 (C-5)
- **임의값 대체 없음.** 결측은 parquet `<NA>`(nullable Int64) 그대로. 0/평균 채움 금지.
- 채점 단계에서 ORACLE 규칙(유효 peer<3 → 페널티)대로 처리한다.

## 알려진 한계 (정직히)
1. **FY2014 재무 미제공**: `fnlttSinglAcntAll` 는 **FY2015부터** 제공(FY2014=013 실측).
   → 2015 스냅샷은 유니버스·산업분류만, 재무는 0. **재무 패널 실질 범위 = 2016~2025.**
2. **산업분류 current-state**: `induty_code` 는 `company.json` 현재값. DART 가 산업분류 개정
   이력을 시점별로 주지 않아 **point-in-time 이 아니다**(100% 채워지나 T 이후 개정분 반영 가능).
   산업분류는 거의 안 바뀌어 룩어헤드 위험은 낮지만, **한계로 명시**한다.
3. **~30% 재무 결측의 성격**:
   - 매출총이익·매출원가는 **서비스·금융업이 원래 손익에 안 적는다**(진짜 없음, 파서 문제 아님).
   - 일부 corp 는 fnlttSinglAcntAll 자체가 013(비표준 XBRL·일부 KONEX).
   - **정정본 보수 처리**: 표준화 재무의 rcept_dt 가 T 이후면(사후 정정) 그 시점엔 미가용으로
     보고 **null**(원본 as-of-T 수치를 API로 못 얻어서). 룩어헤드 대신 결측을 택함.
4. **사업내용 텍스트**: Loop 2(similarity)로 지연. 타당성은 0-E에서 실측(원문 추출 96.7%).
5. **fs_div**: 개별(OFS) 우선·연결(CFS) 대체. 어느 것을 썼는지 `fs_div` 컬럼에 기록.
6. **표본 패널**(위 규모 참조): 전 유니버스 아님. Loop 1에서 확장.
