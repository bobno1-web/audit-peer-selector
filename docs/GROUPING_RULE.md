# GROUPING_RULE — 결과 화면 peer 2그룹 분류 규칙 (Loop WEB-3 · PART 0)

★ 이 규칙은 **표시(화면) 재구성** 전용이다. 엔진 순위·유사도 점수를 **바꾸지 않는다.**
peer 를 두 시각적 그룹으로 나눠 보여줄 뿐, 각 peer 의 원본 순위(rank 1~10)와 유사도(similarity)는
엔진 산출 그대로 표시한다. **그룹 안에서 재번호 금지.** 순위가 1,3,5 처럼 띄엄띄엄이어도 그대로 둔다.

## 1. 분류에 쓰는 실제 필드
엔진 출력 `peers[].top_axes` — 각 peer 의 `rationale`(축별 기여 `industry/scale/mktcap/text/growth`)
에서 **양수 기여 상위 2축**을 한국어 라벨로 변환한 리스트다(`web_engine.py` 산출). 예:
`["규모","시가총액"]`, `["업종","규모"]`, `["규모","사업내용"]`.
라벨↔축 대응은 결과의 `axis_labels`:
`{industry:"업종", scale:"규모", mktcap:"시가총액", text:"사업내용", growth:"성장성"}`.

★ 분류는 **이미 존재하는 `top_axes` 태그만** 본다. peer 의 다른 정보를 새로 추론하지 않는다.
새 점수·새 판정 로직·새 필드를 만들지 않는다.

## 2. 분류 규칙 (이진)
peer 의 `top_axes` 를 기준으로:

- **섹션 1 — "업종·사업이 비슷한 동종기업"**:
  `top_axes` 에 **업종(industry) 또는 사업내용(text)** 라벨이 하나라도 있는 peer.
- **섹션 2 — "규모·시가총액이 비슷한 기업" (부제: 업종은 다를 수 있음)**:
  그 외 — 업종·사업내용 라벨이 **둘 다 없는** peer(규모/시가총액/성장성 위주).

판정식(의사코드, 실제로는 템플릿 Jinja 에서 동일하게 수행):
```
ind = axis_labels.industry   # "업종"
txt = axis_labels.text       # "사업내용"
section = 1 if (ind in top_axes or txt in top_axes) else 2
```

## 3. 경계 케이스
- **`top_axes` 가 빔**(양수 기여 축이 없어 라벨이 0개): 업종·사업 신호가 없으므로 **섹션 2**.
- **섹션 2 가 빔**(모든 peer 가 업종·사업 유사): 섹션 2를 **숨긴다**(섹션 1만 표시).
- **섹션 1 이 빔**(업종·사업 유사 peer 0): "업종·사업내용이 유사한 기업을 찾지 못했습니다" **안내** 후
  섹션 2만 표시.
- 성장성(growth)만 상위인 경우: 업종·사업내용이 아니므로 섹션 2(실데이터상 growth 기여는 작아
  top-2 에 거의 오르지 않음).

## 4. 이 규칙이 하는 일 / 하지 않는 일
- **하는 일**: 유사도 순 나열에서, 규모만 비슷한 다른 업종(예: CJ제일제당의 rank1 삼성중공업)을
  아래 섹션으로 내려 "식품엔 식품이 위에" 보이게 한다.
- **하지 않는 일**: 순위·점수 변경, 재정렬로 순위 숫자 바꾸기, 재번호. 그룹은 시각 분리일 뿐이다.

## 5. 실측 (2022-05-15 스냅샷, out/results/*.json 원본 top_axes 로 분류)
순위 숫자는 전부 엔진 원본. 그룹은 시각 분리일 뿐 재번호 없음.

| 대상 | 신뢰등급 | 섹션1(업종·사업) 원본순위 | 섹션2(규모·시총) 원본순위 |
|---|---|---|---|
| 오뚜기 | HIGH | 1,2,3,4,8 (농심·롯데웰푸드·삼립·오리온·현대지에프홀딩스) | 5,6,7,9,10 (엘에스일렉트릭·KG스틸·에스원·롯데칠성음료·LIG) |
| 강남제비스코 | HIGH | 1,2,3,4,5,7,8,9 (조광페인트·SP삼화·노루페인트 등 도료·화학) | 6,10 (인지컨트롤스·한국공항) |
| 매일유업 | HIGH | 2,3,5,6,7,8,9,10 | 1,4 (삼립·대한제당) |
| CJ제일제당 | LOW | 2,3,5,6,7,8,9 (농심·오뚜기·오리온·롯데웰푸드 등) | 1,4,10 (삼성중공업·현대건설·고려아연) |
| 삼성전자 | LOW | 1~10 전부(업종·사업내용 전자) | (없음 → 섹션2 숨김) |

★ CJ제일제당: rank1 삼성중공업이 섹션2(규모·시총만)로 정확히 내려가고, 섹션1에 식품 peer
(농심·오뚜기·오리온·롯데웰푸드)가 올라온다. 순위 숫자는 엔진 원본 그대로(섹션1=2,3,5,6,7,8,9 /
섹션2=1,4,10). 전체 응집도가 낮아 신뢰등급은 LOW 로 정직하게 유지된다.

## 6. 정직성 정박
- 이 규칙은 `top_axes`(실데이터)만 참조 — 지어낸 필드·추론 0.
- 엔진·채점기·데이터·라우팅·산출물 무변경. 구현은 `templates/result.html`(Jinja)에서만.
- 신뢰등급 배지(`peer_confidence`)·응집도(`peer_cohesion`)는 기존 그대로 표시(그룹핑과 무관).
