# TAG_THRESHOLD — 태그 절대하한 도출 (WEB-10 ①)

> ★ 표시 계층 전용. **엔진·채점기·ORACLE·순위·유사도 점수 불변.** 이 문서는 "사업내용 유사" 등
> 축 라벨(top_axes)에 붙는 **절대 하한**을 **데이터에서 도출**한 근거다(하드코딩 금지).
> 재현: `python scripts/derive_tag_floor.py [--verify-cj]` → `runs/web10_tag_floor/`.

## 1. 문제 (T1) — 절대 하한이 없어 상투어 겹침이 "유사"로

표시 로직(`web_engine.query`)은 각 peer 의 rationale(축별 가중 기여 `w·sim`) 상위 2축(값>0)에
한국어 라벨을 붙였다. **절대 하한이 없다.** 그래서 텍스트(사업내용) 축은 사업보고서 상투어
("제조·판매를 영위", "사업의 내용", "당사는 …") 겹침만으로도 무관 산업 쌍의 코사인이 0 근처로
죽지 않아, 식품↔전선·철강·전자 같은 **오판정**이 "사업내용 유사"로 붙었다.

실측(현재 2025 스냅샷, 원 텍스트 코사인):

| 타겟 | 잘못 붙은 peer | text 코사인 | 실제 |
|---|---|---|---|
| CJ제일제당 | 삼성전기 | 0.375 | 전자부품(무관) |
| CJ제일제당 | 두산에너빌리티 | 0.365 | 발전설비(무관) |
| 오뚜기 | 대한전선 | 0.345 | 전선(무관) |
| 오뚜기 | KG스틸 | 0.446 | 철강(무관) |

이들은 rationale 상위 2축에 text 가 들어 "사업내용 유사" 라벨을 받고, 그룹1(비교 가능한
동종기업)에까지 올라왔다(`docs/GROUPING_RULE.md`: top_axes 에 업종/사업내용 있으면 그룹1).

## 2. 하한 도출 — 데이터 유도 (T2, T3)

**널(null) 정의.** 회사 쌍 중 **KSIC 2자리(대분류)가 서로 다른 쌍** = 사업이 무관한 쌍.
이들의 텍스트 코사인은 (정의상) 실제 사업내용 겹침이 아니라 **상투어 겹침**만 반영한다.

**하한 = 널 분포의 p99.** "무관 산업 쌍이 상투어만으로 우연히 도달하는 코사인을 상위 1% 이내로만
초과" = 통계적으로 유의한 **실제** 텍스트 겹침으로 본다. 분위수(p99)는 방법으로 고정하고, **값은
dev 데이터에서** 나온다(임의의 0.5 를 찍지 않는다).

**dev 만 사용(holdout 봉인).** dev 연도 2016–2022 각각에서 그 해 유니버스(텍스트벡터·KSIC 보유
기업)의 상삼각 쌍 코사인을 같은-KSIC2 / 다른-KSIC2 로 분리, 다른-KSIC2(=널)의 p99 를 구한다.
텍스트 벡터는 엔진이 실제 쓰는 `section_vectors.npz`(문자 3-gram TF-IDF, L2정규화 → 내적=코사인).

**결과 (`runs/web10_tag_floor/derivation.json`):**

| dev year | 기업수 | 널(다른KSIC2) p50 | 널 p99 | 같은KSIC2 p50 | AUC(같은>다른) |
|---|---|---|---|---|---|
| 2016 | — | 0.18 | **0.5202** | ~0.30 | 0.653 |
| 2017 | — | 0.18 | **0.5176** | | |
| 2018 | | | **0.5174** | | |
| 2019 | | | **0.5171** | | |
| 2020 | | | **0.5204** | | |
| 2021 | | | **0.5228** | | |
| 2022 | | | **0.5245** | | |
| **평균** | | | **0.5200** | | **0.653** |

- **하한 값 = 0.52** (7개 dev 연도 널 p99 평균, **표준편차 0.0027**).
- 임의값이 아니다: p99 는 방법 고정, 0.52 는 데이터에서 나온 평균. 연도 표준편차 0.0027 로 매우
  안정 → **2025/2026 스냅샷에도 재유도 없이 타당**(H-2 에서 재확인).

## 3. 판별력 — 하한이 실제 산업일치와 상관되는가 (T3)

**AUC = 0.653** = P(같은-KSIC2 쌍 코사인 > 다른-KSIC2 쌍 코사인). 0.5=무판별, 1.0=완전판별.
→ 텍스트 코사인은 **실제 산업 일치와 유의하게 상관**되지만(우연 0.5 초과), 상투어 때문에 널
분포의 상위 꼬리(p99)가 0.52까지 올라온다. **바로 이 상투어 꼬리를 잘라내는 것이 하한의 역할**이다.

## 4. 축별 하한 필요성 검토 (T4) — text 만 하한, 나머지는 불필요

| 축 | 하한? | 근거 |
|---|---|---|
| **text(사업내용)** | **필요, 0.52** | 상투어 널이 p50=0.18·**p99=0.52** — 0 근처로 안 죽음. 상투어 꼬리 절단 필요. |
| industry(업종) | 불필요 | sim 이 KSIC 접두매칭 이산값(1.0/0.5/0). sim>0 이면 이미 실제 산업 일치(범주형). 상투어 baseline 없음. |
| scale(규모) | 불필요 | exp(−거리) 커널 — 무관 쌍 sim≈0(널 median 0.00). |
| mktcap(시총) | 불필요 | exp(−거리) 커널. 널 median 0.215이나 이는 실제 로그시총 근접이지 상투어 아님. 가중치 최저군. |
| growth(성장) | 불필요 | exp(−거리) 커널 — 무관 쌍 sim≈0. 가중치 최저. |

→ **상투어 baseline 문제는 텍스트 축에만 있다.** 다른 축은 유사도가 무관 쌍에서 0으로 죽으므로
"상위 2축 & 값>0" 규칙이 이미 충분. `config/tag_thresholds.json.axis_floors = {"text": 0.52}`.

## 5. 로직 적용 (T5) — 라벨만, 순위·점수 불변

`web_engine.query`: peer 의 rationale 상위 2축(값>0)에 라벨을 붙이되, 그 축에 하한이 있으면
**원(raw) 축 유사도**(`sim_c`, 가중 전)가 하한 미만이면 라벨을 **제거**한다. rationale 은 `w·sim`
이므로 하한 비교는 raw `sim` 에 한다. **peer 선정·순위(order)·유사도(total) 는 손대지 않는다** —
`top_axes` 리스트만 조건부. 하한으로 라벨이 다 빠지면 그 peer 는 그룹 규칙상 자연히 그룹2
(규모·시총 유사)로 내려간다(태그 삭제가 아니라 미부여, `docs/GROUPING_RULE.md` 정합).

## 6. 검증 (T6, T8) — 오판정 제거 + 진짜 유사 보존

`scripts/derive_tag_floor.py --verify-cj` + 웹 질의 before/after (2025 스냅샷):

**오판정 제거 (T6):** 삼성전기(0.375)·두산에너빌리티(0.365)·대한전선(0.345)·KG스틸(0.446) 모두
< 0.52 → "사업내용 유사" 라벨 **사라짐**. 이들은 top_axes 가 [규모]만 남아 그룹2(규모·시총 유사)로
내려간다 = 정직한 배치(사업이 아니라 규모가 비슷한 것).

**진짜 유사 보존 = 과도하지 않음 (T8):** 오뚜기↔**동원F&B 코사인 0.576 ≥ 0.52** → "사업내용 유사"
**유지**. 순수 식품–식품 사업내용 겹침은 하한을 넘어 살아남는다. 즉 하한은 전선·철강 오판정을
죽이면서 실제 식품–식품 겹침은 지킨다(같은 오뚜기 목록에서 동원F&B 유지 vs 대한전선/KG스틸 제거).

**주의(정직):** CJ제일제당은 **자신의 텍스트가 지주·복합 상투어에 가까워** 식품 peer 와의 코사인도
0.26–0.32 로 낮다(현재 텍스트가 대부분 2016 제출본이라 낡음). 그래서 CJ 의 식품 peer 는 하한 적용시
"사업내용" 라벨을 잃고 **"업종" 라벨로 그룹1 유지**된다(peer 자체는 순위·표시 그대로). 이는 하한이
과도한 게 아니라 **낡은 텍스트가 겹침을 입증 못 하는 것** — WEB-10 ②(2025 사업보고서로 텍스트
최신화)가 이 부분을 개선한다(H-2/H-3에서 재확인). 안 비슷한 걸 "유사"라 부르지 않는다는 원칙(①-D)에
부합: 근거(텍스트 겹침) 없으면 라벨을 안 붙인다.

## 6.5 ★ H-2: 하한은 '벡터 공간별'로 유도 (프레시 2025 텍스트, WEB-10 ②)

②에서 2026 스냅샷에 FY2025 '사업의 내용'을 최신화(`section_vectors_2026.npz`, 2759곳)하자 하한
검증에서 **중요한 발견**: 프레시 2026 벡터는 **vocab/IDF 가 dev 공유공간과 달라 절대 코사인 척도가
다르다**. 같은 METHOD(cross-KSIC2 널 p99)를 프레시 2026 공간에 적용하면:

| 벡터 공간 | 널 p50 | **널 p99 = 하한** | AUC | 적용 스냅샷 |
|---|---|---|---|---|
| dev 공유(section_vectors.npz) | 0.18 | **0.52** | 0.653 | 2022/2024/2025 |
| 프레시 2026(section_vectors_2026.npz) | 0.18 | **0.49** | **0.680** | 2026 |

dev 하한 0.52 를 프레시 2026 에 그대로 쓰면 **과도**했다: 진짜 F&B peer **오뚜기↔롯데칠성음료
(0.506)** 를 제거. 프레시 공간의 정당한 하한은 그 공간의 널 p99 = **0.49**. (AUC 0.680 > 0.653 —
프레시 텍스트가 산업신호를 더 강하게 담는다.)

→ **METHOD(널 p99)는 동결, VALUE 는 공간별 유도.** 엔진 `similarity/run.py::_robust_scale` 이 스냅샷마다
IQR 을 재유도하는 것과 **동형** — 임의 조정이 아니라 각 벡터공간의 데이터에서 나온 값이다. 하한은
벡터파일에 결속한다: `section_vectors_{year}.floor.json`(사이드카, no-false-provenance). 공유공간을 쓰는
연도는 config 의 0.52.

**H-3 프레시 2026 재확인(floor=0.49):**
- 오뚜기: KG스틸(.421)·대한유화(.373)·에스원(.334) "사업내용" 제거(이종), **롯데칠성음료(.506) 유지**(진짜 F&B).
- CJ제일제당: 삼성전기·SK가스·대우건설·아시아나 등 이종 제거. 식품 peer 는 순위·"업종" 라벨로 유지되나
  "사업내용"은 미부여 — CJ 텍스트가 지주·복합(식품+바이오+…) 광범위라 특정 식품사와의 텍스트 겹침이
  약함(정직: 근거 약하면 라벨 안 붙임). **잔여 한계:** CJ↔CJ대한통운(.499)은 **같은 그룹** 상투어로 경계
  통과 — 산업레벨 널이 못 잡는 그룹레벨 상투어(정직히 기록, 향후 개선 여지).

## 7. 재현

```
python scripts/derive_tag_floor.py --verify-cj --year 2025
# → runs/web10_tag_floor/derivation.json   (연도별 널 p99·AUC, floor=0.52)
# → runs/web10_tag_floor/verify_cj.json     (CJ/오뚜기 오판정 원 코사인)
```

값은 `config/tag_thresholds.json` 에 스탬프. 정직성: 이전 config 의 `source_script` 는 삭제된
scratchpad 스크립트를 가리켜 재현 불가였다 → **커밋 스크립트 `scripts/derive_tag_floor.py` 로 결속**
(no-false-provenance).
