# EMBEDDING_SCOUT — 신경망 임베딩 효과 정찰 (Loop 4 PART 0)

★ **무료 정찰. 임베딩/외부 API 호출 = 0.** 캐시된 데이터(targets·industry·section_text·L3 peers)만.
원자료: `runs/2026-07-15_loop4_scout/scout.json`. 스크립트: `scripts/loop4_embedding_scout.py`.
목적: 신경망 임베딩에 **돈 쓸 가치**가 있는지 — 그것이 파고들 '틈'이 실제로 있는지 공짜로 판정.

> 이건 **분석(정찰)**이다 — 엔진이 아니다. 분석자는 targets 를 봐도 된다(채점기와 같은 지위).
> 엔진(SIM)은 여전히 as_of features 만 쓴다. 룩어헤드/누출 아님.

## 0-1. 산업코드 해상도 — **코드는 뭉뚱그린다 (틈 있음)**
연도당 산업코드 ≈ **505개**. 기업이 몇 개짜리 코드에 묶이나:

| 코드 크기 | 1개 | 2–5개 | 6–20개 | 21개+ |
|---|--:|--:|--:|--:|
| 그 크기 코드에 속한 기업 비중 | 8.6% | 24.2% | 35.9% | 31.2% |

- **핵심 지표 = eta²** (일원분산분석: 산업코드가 채점 비율의 분산을 얼마나 설명하나. 0=전혀, 1=완전):

| 비율 | eta² (코드가 설명하는 분산) |
|---|--:|
| 매출총이익률 | 0.516 (절반은 설명) |
| **영업이익률** | **0.136 (거의 못 설명)** ★ 개선표적 |
| 재고자산회전율 | 0.355 |
| 매출채권회전율 | 0.237 |
| **평균** | **0.311** |

→ **산업코드는 채점 비율을 평균 31%밖에 설명하지 못한다.** 특히 **영업이익률은 14%** — 같은 산업코드
안에서도 영업이익률이 크게 흩어진다. **원칙적으로는 텍스트/세부축이 파고들 틈이 크다** (특히 영업이익률).

## 0-2. 얕은 텍스트 신호의 성격 — **틈을 '붐비는 코드'에서 메우지 않는다**
L3의 텍스트 축(가중치 0.31)을 껐다 켜서 케이스별 도움(=APE(text off)−APE(text on), +면 개선)을 측정
(n=11,536 타겟, 커밋된 L3 peers 재사용):

| 지표 | 값 | 해석 |
|---|--:|---|
| 텍스트 도움(전체 평균) | **+0.030** | 작지만 양(+) — Loop 3 결과와 일관 |
| corr(도움, 코드 붐빔) | **−0.008** | ≈0 — 붐빌수록 더 돕지 **않음** |
| corr(도움, 코드 내 이질성) | **−0.001** | ≈0 — 이질적일수록 더 돕지 **않음** |
| 도움: 붐비는 코드 | **−0.097** | 오히려 소폭 **해**(노이즈) |
| 도움: 희소 코드 | **+0.144** | 텍스트는 **희소 코드에서** 돕는다 |

→ ★ **가설과 반대다.** 텍스트의 이득은 "같은 코드에 몰린 다수 기업을 텍스트로 구별"(붐빔 해소)에서
오지 **않는다.** 오히려 **코드에 peer가 부족한 기업**에서, 텍스트가 **다른 코드의 사업 유사 기업**을
찾아줄 때 온다. 신경망 임베딩의 강점(의미적 구별)이 빛날 자리(붐비는 이질 코드)에서는 얕은 TF-IDF가
이미 별 이득이 없고 임베딩이 파고들 표면적이 작다.

## 0-3. L3 최악 20 타겟 육안검사 — **큰 오차는 축으로 못 고친다**
원인 분류(자동 신호 + 육안): **(a) 산업코드 오묶음(텍스트/임베딩 여지) / (b) 본질적 특이(어떤 축도
못 고침) / (c) 데이터**.

| 원인 | 건수 | 대표 진단 |
|---|--:|---|
| **(b) 본질적 특이** | **16 / 20** | 타겟 비율 actual≈0 → **APE 폭발**, 또는 peer 회전율 불안정(작은 분모→폭증) |
| (a) 산업코드 오묶음 | 3 / 20 | peer 흩어짐(같은코드 소수·텍스트코사인 낮음) → 더 나은 텍스트 여지 |
| (c) 데이터 | 1 / 20 | 텍스트 결측(no_titles) |

- ★ **16/20이 "actual ≈ 0"**: 하이즈항공(매출총이익률 actual=−0.0), 서부T&D(영업이익률 −0.0001),
  현대위아(−0.0001), 두산테스나(−0.0005), 남양유업(−0.0002)… **peer 텍스트 코사인이 0.6~0.8로 좋아도**
  (두산테스나 0.83, 현대위아 0.66) 타겟 자신의 비율이 0 근처 이상치라 상대오차가 폭발한다.
  → **어떤 peer 축(텍스트·임베딩·부문)으로도 못 고친다.** 임베딩은 "사업 비슷한 peer"를 더 잘 찾을 뿐,
  타겟의 0 근처 실적이나 회전율 수치 불안정을 고치지 못한다.
- (a)로 분류된 3건조차 driver 가 회전율(peer의 작은 재고/매출채권 → 예측 폭증)이라 **표현 개선보다
  비율 안정화 문제**에 가깝다.

## ★ 판정: 신경망 임베딩 유의미 가능성 = **낮음~중간 (LOW–MEDIUM)**

| 근거 | 임베딩에 유리 | 임베딩에 불리 |
|---|---|---|
| 0-1 산업코드 해상도 | 코드가 뭉뚱그림(eta² 0.31, 영업이익률 0.14) → 원칙적 틈 O | — |
| 0-2 텍스트 신호 성격 | 텍스트가 양(+)의 이득 | 이득이 **작고(+0.03)**, 임베딩 강점 자리(붐빔·이질 코드)와 **무상관** |
| 0-3 최악 케이스 | (a) 3/20 은 여지 | **16/20 은 APE폭발·회전율불안정 → 어떤 표현도 못 고침** |

**결론(정직):**
- **틈은 있으나(코드가 성기다) 임베딩이 그 틈을 크게 메울 근거는 약하다.** 얕은 텍스트조차 이득이 작고
  (+0.03), 그 이득이 임베딩이 잘하는 자리(붐비는 이질 코드 구별)가 아니라 **희소 코드의 교차매칭**에서
  온다 — 거기선 TF-IDF도 이미 통한다.
- **집계 지표(중앙 APE)의 상한을 실제로 누르는 건 표현 품질이 아니라 지표의 꼬리 행태**(0 근처 실적 →
  APE 폭발, 회전율 수치 불안정)로 보인다. 이건 임베딩으로 안 풀린다.
- ★ **권고(잠정, PART 4에서 L4 결과와 합산해 확정):** **아직 돈 쓰지 마라.** 이번 루프의 무료 축
  (부문별매출·성장)을 먼저 소진하고, 임베딩은 **낮은 우선순위 옵션**으로 남긴다. 임베딩이 승리조건
  (0.433)을 깨리라 기대할 근거는 이 정찰에 없다.

> 최종 임베딩 권고는 `docs/SHOWDOWN_L4.md` 에서 이 정찰 + L4 실측(무료 축이 실제로 얼마나 도왔나)을
> 합산해 확정한다.
