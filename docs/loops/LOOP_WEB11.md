# LOOP_WEB11 — 사용자 친화 문구 정리 + Footer 전 페이지 통일

## 프로젝트 (확인)
`동종업계탐색기_클로드` (PeerLens / peer-selection). app.py·templates.
★ DartLens·뉴스시그널 참조 지시면 다른 프로젝트 — 중단.

## 의미
사용자가 이해하기 어려운 전문 용어(응집도·평가시점·L6 엔진·point-in-time)를 쉬운 표현으로.
Footer 를 전 페이지 동일하게 통일. ★ 엔진·채점기·데이터·결과 로직·순위·점수·신뢰등급 판정 무변경 — 화면 문구만.

## PART 1. 신뢰도 설명 쉽게 (결과 페이지)
- 1-1 현재 "신뢰도 보통 · 동종기업 응집도(상위 유사도 평균) 0.46 기준 등급" → "응집도 0.46" 전문어 제거.
- 1-2 "신뢰도 [등급]" + 왜 그 등급인지 쉬운 설명:
  - 높음: "신뢰도 높음 — 비교할 동종기업이 뚜렷합니다"
  - 보통: "신뢰도 보통 — 비슷한 회사가 어느 정도 있어요"
  - 낮음: "신뢰도 낮음 — 비슷한 회사가 마땅치 않아 참고만 하세요"
- 1-3 "응집도 0.46 기준 등급" 숫자·전문어 표기 삭제. 등급 판정 로직(내부 grade())·배지색(앰버·중립, Web-8) 유지.
  LOW hint 의 "동종기업 응집도가 낮습니다" 도 전문어 → "비슷한 회사가 마땅치 않습니다" 로.

## PART 2. 평가시점 삭제 (결과 페이지)
- 2-1 현재 "회계연도 2025 · 평가시점 2026-05-15" → 사용자 "오늘 7월인데 왜 5월?" 혼란.
- 2-2 "평가시점 {as_of}" 표기 삭제, "회계연도 {fiscal_year}"만 남김. point-in-time 내부 as_of 로직 무변경(표기만 제거).

## PART 3. 결과 하단 문구 삭제
- 3-1 결과 하단 "검증된 L6 엔진(similarity_l6) 실시간 산출 · {as_of} point-in-time 스냅샷 · 신뢰등급·근거는
  엔진 성능을 바꾸지 않는 표현 계층입니다" 완전 삭제(전문어·중복). → footer 를 통일 PeerLens 문구로 교체.
  (정직성: /example 미리저장 결과는 상단에 조용한 표시 유지 — 라이브와 구분, no-false-provenance.)

## PART 4. Footer 전 페이지 통일
- 4-1 다른 프로젝트 문구("© 2026 뉴스시그널 …") 확인 → 리포지토리 전역 grep "뉴스시그널" 0건(애초 없음). 잔존 없음 확인.
- 4-2 전 페이지(랜딩/키/검색/결과) footer 동일: "© 2026 PeerLens · 동종기업 선정·비교 로컬 도구".
  상단 0.5px #efece5(--line) 구분선, 가운데, 11px/#b0aca4(--faintest) — Web-8 톤.
- 4-3 base.html 공용 footer 로 통일(index/key/search 상속 = 각 1개). result.html 은 standalone 이라 동일 문구·스타일 inline(1개). 중복 0.

## 구현 (표시 문구만)
- base.html: body 에 `<footer class="site">© 2026 PeerLens · …</footer>` 추가(기존 `footer.site` CSS 재사용, 미사용→사용).
- result.html: 메타 줄 "· 평가시점 {as_of}" 삭제 / 신뢰도 group 의 cohesion span → 등급별 쉬운 설명 / LOW hint 전문어 순화 /
  하단 footer 문구 → 통일 PeerLens(라이브·예시 공통), 예시 표시는 본문 상단 조용한 note 로 이동.
