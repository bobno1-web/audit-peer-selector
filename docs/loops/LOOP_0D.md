## PART D. 비상장 스파이크 — GO/NO-GO (핵심)
목표: 감사보고서에서 재무제표를 구조화 데이터로 뽑을 수 있는가.
- D-1. 감사보고서 실제 제출 비상장 100개 무작위 추출. 대상 연도 2020~2022.
- D-2. 추출 경로 3가지 전부 시도, 각 성공률 측정. 경로1: 재무 원본파일(XBRL) API. 경로2: 감사보고서
  첨부 XBRL 인스턴스. 경로3: 원문(HTML/텍스트) 표 파싱. ★ 경로3은 실제 파서로 표본 20개+ 돌려 숫자를 낸다.
- D-3. 8개 계정별 추출 성공률: 매출액/매출원가/매출총이익/영업이익/재고자산/매출채권/총자산/종업원수.
- D-4. 결측 편향 조사(추출 실패가 특정 산업/규모에 쏠렸는가).
- D-5. 사업내용 텍스트 대체재 조사(감사보고서 주석의 사업설명 존재/수준).
- 출력: docs/SPIKE_UNLISTED.md (경로별·계정별 실측, 결측편향, 텍스트, 추측/실측 구분).

## PART E. GO/NO-GO 판정 (기준 고정 — 결과 전 확정)
- GO(비상장 포함): 8개 계정 **전부 ≥ 70%** AND 결측이 특정 산업/규모에 심하게 쏠리지 않음.
- NO-GO(상장사만): 미달 → PLAN.md 유니버스 "상장사만" 축소 + 구조적 한계 명시 + DECISIONS.md 기록.
- 애매하면 NO-GO. 기준을 늘려 잡지 마라. DECISIONS.md에 D-006으로 기록.
