# LOOP_WEB2 — 랜딩 디자인 개선 (여백·폰트·색 절제)

## 의미
현직자 피드백: 현재 랜딩이 "AI 티가 난다". 원인 ① 배경 전체가 크림(주황빛)이라 색 과함
② 여백 부족으로 빽빽함 ③ 폰트가 기본 느낌.
참고 톤: 네이버 웨일 시작 페이지 — **순백 배경 + 거대한 헤드라인 + 넉넉한 여백 + 색은 카드 포인트로만**.

## ★ 절대 원칙 — 표현 계층만
디자인(templates/*.html)만 수정한다. **엔진·채점기·데이터·라우팅·산출물 무변경.**
- 색은 "가득 차면" 안 되고 "콕콕 포인트"여야 한다(웨일이 파랑을 배경 아닌 카드에만 쓰듯).
- 실제 데이터만 표시(지어낸 필드 0), 키 미저장, holdout 미사용은 Web-1 그대로 유지.

## 수정 사항 (요약)
1. **배경: 크림 → 순백(#FFFFFF).** 크림(#FBF0E8) 배경 전면 제거. 코랄(#D85A30)은 포인트로만
   (로고·강조 단어·시작하기 버튼·3단계 아이콘·링크). 카드/아이콘의 아주 연한 코랄 틴트만 허용.
2. **예시 회사 칩 UI 제거.** "키 없이 예시로 보기" + 회사 칩 뭉치 삭제. 단 예시 라우트·데이터는
   유지(입력창에 회사명 직접 입력으로 접근 가능). 못 찾을 때 뜨는 오타 후보 칩(에러 UX)은 유지.
3. **3단계 카드 유지 + 확대.** 01 입력 / 02 비교 / 03 순위 유지. 패딩·간격·아이콘 크게, 여백 넉넉히.
   흰 카드 + 얇은 테두리 + 부드러운 그림자. 왼쪽 선 금지.
4. **Pretendard 폰트.** CDN(jsdelivr orioncactus/pretendard) 로드, 본문 400 / 제목 600~800.
   로드 실패 시 system-ui, sans-serif 폴백.
5. **여백·타이포 위계.** 헤드라인 확대·임팩트, 히어로 상하 여백 대폭 확대, 섹션 구분은 선이 아니라 여백.
6. **결과 페이지: 폰트만 통일.** 레이아웃·배지·안내박스 그대로, Pretendard만 동일 적용.

## 유지 (변경 금지)
헤드라인 "이 회사, 무엇과 비교할까요?"(강조 코랄)·부제·CTA "시작하기"·3단계 문구·코랄=포인트색·
신뢰등급 배지 색(초록/노랑/회색)·실제 데이터만·엔진/채점기 무변경·키 미저장·holdout 미사용.

## 금지
배경 전체를 색으로 채우기 / 카드 왼쪽 세로선·회색 배경·무거운 테두리 /
엔진·채점기·데이터·라우팅·산출물 수정 / 지어낸 필드 추가.

## 완료 조건 V1~V14
- **V1** 페이지 배경 순백(#fff), 크림 배경 제거
- **V2** 코랄은 포인트로만(로고·강조·버튼·아이콘), 배경색으로 안 씀
- **V3** 예시 회사 칩 UI 제거 (강원에너지 등)
- **V4** 예시 조회 기능은 입력으로 접근 가능(라우트 유지)
- **V5** 3단계 카드 유지, 더 크고 여유롭게(패딩·간격·아이콘 확대)
- **V6** Pretendard 폰트 로드, 폴백 처리
- **V7** 헤드라인 확대, 히어로 여백 확대
- **V8** 결과 페이지에 Pretendard 통일, 레이아웃 유지

- **V10** 헤드라인·부제·CTA·3단계 문구 유지
- **V11** 신뢰등급 배지 색 유지
- **V12** 엔진·채점기·데이터·라우팅·산출물 미변경 (git diff 확인)
- **V13** 지어낸 필드 0, 키 미저장, holdout 미사용
