# LOOP_WEB12 — 유사도 점수 숫자 제거 (순서 배열로만)

## 프로젝트 (확인)
`동종업계탐색기_클로드` (PeerLens / peer-selection). app.py·templates.
★ DartLens·뉴스시그널 참조 지시면 다른 프로젝트 — 중단.

## 의미
유사도 점수 숫자(0.70·0.46 등)를 화면에서 제거. 사용자는 "0.46"의 의미를 모르고, 이 점수는
절대 기준이 없다(상대 순서만 의미). 숫자를 빼고 "위에서부터 비슷한 순서"로만 배열 → 정도를
단정하지 않고 순서로만 암시. ★ 엔진·채점기·순위 로직·유사도 계산 무변경. 화면 표시만.

## 배경
- 유사도 0.46은 "46% 유사"로 오해 가능. 절대 기준 없음 — "이 회사들 중 상대 순서"일 뿐.
- 순위 숫자(1,2,3)는 이미 제거됨. 유사도 숫자도 같은 이유로 제거.
- "가장 비슷" 라벨은 "무슨 기준?" 공격 가능 → 안 씀. 순서 배열만으로 "위=더 비슷" 자연 이해.

## PART 1. 유사도 숫자 제거
- 1-1 결과 각 peer 행에서 "유사도 0.XX" 제거(우측 큰 숫자 23px/700 + "유사도" 라벨).
- 1-2 ★ 내부 정렬은 유사도 내림차순 그대로. 화면 순서 = 유사도 높은 순(숫자만 숨김). 정렬 로직 무변경.
- 1-3 peer 행에 남는 것: 회사명 + 태그. 숫자 자리는 깔끔한 여백(억지 채움 X).
- 1-4 ★ 라벨·등급·별표·막대 넣지 않음. 순수 순서만("가장 비슷"·★·정도막대 전부 없음).

## PART 2. 유지
- 2-1 태그(업종/규모/사업내용 유사) 유지(사실).
- 2-2 그룹 구분(업종·사업 유사 / 규모·시총만) 유지.
- 2-3 신뢰등급(높음/보통/낮음 + 쉬운 설명, Web-11) 유지. ★ 신뢰등급=절대수준, 순서=상대위치.
- 2-4 접이식·정직안내(업종유사 0)·태그 하한(Web-10)·최신 데이터(2025) 유지.
- 2-5 PeerLens 디자인(Web-8)·footer 통일(Web-11) 유지.
