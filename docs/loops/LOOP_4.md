## 이번 루프의 의미
Loop 3에서 텍스트가 살아났다(0.31). 이번엔 돈 드는 신경망 임베딩 전에
무료 축 두 개를 먼저 넣고, 임베딩이 효과 있을지 공짜로 정찰한다.
★ 원칙: 무료 방법을 다 써본 뒤, 정말 필요할 때만 유료 임베딩.
★ holdout(2023~2025)은 이번에도 열지 않는다.

## 순서
  PART 0(임베딩 정찰, 무료) → PART 1(부문별매출) → PART 2(기업연령)
  → PART 3(재조합) → PART 4(결판) → PART Z

## PART 0. 임베딩 효과 정찰  ★ 돈 쓰기 전 무료 확인
═══════════════════════════════════════

목표: 신경망 임베딩에 만 원 쓸 가치가 있는지 공짜로 판정한다.

0-1. 산업코드 해상도 측정
  - 각 산업코드에 몇 개 기업이 묶여 있나 분포를 낸다
  - ★ 같은 산업코드 안에서 채점 4비율의 이질성(분산)이 큰가?
    크면 = 산업코드가 뭉뚱그린다 = 텍스트/세부축이 파고들 틈 있음
    작으면 = 산업코드가 이미 촘촘 = 추가축 효과 제한적

0-2. 얕은 텍스트 신호의 성격 분석
  - Loop 3에서 텍스트 가중치 0.31이 나왔다.
  - ★ 텍스트가 기여한 케이스가 "산업코드가 애매한 기업"에 몰려 있나?
    (같은 산업코드 다수 기업 중 텍스트로 구별된 사례)
  - 몰려 있으면 = 신경망 임베딩이 더 파고들 여지 큼

0-3. 오분류 사례 육안 검사
  - L3가 크게 틀린 타겟 20개의 peer를 열어본다
  - 원인 분류: (a)산업코드가 엉뚱하게 묶음 → 텍스트/임베딩이 고칠 여지
              (b)그 기업이 본질적으로 특이 → 어떤 축도 못 고침
              (c)데이터 문제

0-4. docs/EMBEDDING_SCOUT.md
  - 산업코드 해상도 판정
  - 텍스트 기여의 성격
  - 오분류 (a)/(b)/(c) 비율
  - ★ 판정: 신경망 임베딩이 유의미할 가능성 (높음/중간/낮음) + 근거
  ★ 이건 정찰이다. 임베딩을 실제로 하지 않는다. API 호출 0.

## PART 1. 부문별 매출 축  ★ 임베딩 상위호환 후보
═══════════════════════════════════════

가설: 사업보고서의 부문별 매출(예: 반도체45%+가전20%)을 숫자로 쓰면
      "사업 포트폴리오가 비슷한 회사"를 텍스트보다 정밀하게 찾는다.

1-1. 부문별 매출 추출
  - 사업보고서/주석의 부문(segment)별 매출 공시를 파싱
  - 각 기업을 부문 비중 벡터로 표현 (예: [반도체0.45, 가전0.20, ...])
  - ★ 부문 분류 체계가 기업마다 다르므로, 공통 축으로 정규화하는 방법 명시
  - 추출 성공률 보고. 실패는 결측(임의대체 금지).

1-2. ★ 정보 차단벽 재확인
  부문별 "매출 비중"은 허용 (사업 구성 정보).
  단, 부문별 "이익률"은 채점 비율과 겹칠 수 있으니 쓰지 마라.
  ★ 이 축이 타겟의 채점비율을 재구성 가능하게 하는지 test_no_leakage로 검사.

1-3. docs/SEGMENT_EXTRACTION.md: 성공률 + 정규화 방법 + 품질 표본
