## PART 1. 항진명제 차단  ★ 여섯 번째 실수 봉쇄

상폐=제출중단이므로 잔존율은 입력과 무관하게 영구 0%. 논리적으로 실패 불가능.
0-H의 4검사·코드판정·소스실재·자기기만경고 중 어느 것도 못 잡았다.
1-1. gate_design.md 5번째 검사 [비자명성] 추가 (논리적으로 실패 가능한가; 상수면 자명).
1-2. test_gate_necessity 확장 — 항진명제 기계 탐지(소스 교란 시 measured 불변이면 FAIL,
     또는 falsifiability_test 필드 필수). ★ D-016 ①이 FAIL 나는지 실증·보고.
1-3. DECISIONS D-017: ① 항진명제 판명, 판정 근거에서 제외. PASS는 ②만으로 정당.

## PART 2. judge 측정값 검산 강제  ★ 더 위험한 구멍
【진단】judge가 gate 파일 measured 를 그대로 신뢰·재측정 안 함. 측정값 통제 에이전트가
임계 통과값을 써넣으면 PASS. "approve 제거→코드판정"이 새 구멍을 열었다.
2-1. gate.py judge 수정 — runs/ 원자료에서 measured 자가 재집계·대조, 불일치 시 판정 거부.
     ★ 재집계 로직은 측정 스크립트와 독립 구현(같은 버그 공유 금지).
2-2. gate 파일 measurement_provenance 필수화(각 값의 원자료·집계 경로). 재집계 불가 시 거부.
2-3. test_gate_measurement_integrity.py 신규 — 위조 measured → 원자료 대조로 차단 실증.
     원자료 없으면 거부. ★ 위조 통과 시도가 막히는지 실증·보고.
2-4. gates-are-binding.md 강화 — "측정값 통제자≠판정 통제자, judge가 원자료 재집계".
