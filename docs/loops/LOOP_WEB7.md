# LOOP_WEB7 — 랜딩 네이버웨일 스타일 재구성 + 3페이지 플로우

## 프로젝트 (확인)
`동종업계탐색기_클로드` (peer-selection). app.py·templates·engines/similarity. ★ DartLens·Streamlit·
재무비율 이상징후는 다른 프로젝트 — 건드리지 않음.

## 의미
현직자 피드백: 랜딩이 "옹졸하고 중앙에 몰려 시원함이 없다". 네이버 웨일 시작 페이지처럼 시원하게
재구성 + 입력을 별도 페이지로 분리. ★ 표시 계층만. 엔진·채점기·데이터·결과 화면 로직 무변경.
결과 표시 규칙(업종/규모 그룹·순위무조작·정직안내·신뢰등급)은 그대로 유지.

## PART 1. 랜딩(/) — 네이버 웨일 스타일
- 로고 좌상단(돋보기 + "동종기업 선정기", 왼쪽 정렬, 상단바 왼쪽 끝).
- 히어로 중앙: 헤드라인 "이 회사, 무엇과 비교할까요?"("무엇과 비교" 코랄) 크게, 위아래 여백 대폭.
  부제 유지. ★ 입력창(회사·키) 랜딩에서 제거 → "시작하기"(코랄) 버튼만 → /key.
  넓게: 중앙 몰림 해소, max-width 너무 좁게 X.
- 3단계 소개 카드(도구 소개, A안): 01 비슷한 회사를 찾아줍니다 / 02 왜 골랐는지 보여줍니다 /
  03 얼마나 믿을지 알려줍니다. 흰 카드+얇은 테두리+그림자+아이콘, 왼쪽선 금지, 크고 넓게.
- ★ 하단 회색 안내문 제거(키 보안 안내는 /key 로 이동).

## PART 2. 키 입력(/key)
- OpenDART 키 입력(선택, 비우면 pre-built 스냅샷). 키 보안 안내 배치. "다음" → /search.
- 로고 좌상단, 웨일 톤. ★ 키 세션 전달만, 저장 0(쿠키·파일·로그·디스크 X — hidden 폼 전달).

## PART 3. 회사·연도(/search)
- 회사명(placeholder "예: 오뚜기") + 연도 선택(실데이터 available years). "동종기업 찾기" → 결과.
- 로고 좌상단, 웨일 톤.

## PART 4. 진짜 라우팅
- / → /key → /search → /query(결과). 뒤로가기·직접 URL 자연스럽게. 입력값 세션 내 전달(저장 0).
- 키·회사·연도는 폼(POST hidden)·쿼리로 전달. 서버 저장·쿠키 세션 미사용.

## 유지/금지
- 결과 페이지(result.html) Web-3/4 그대로. 엔진·채점기·데이터·최신 스냅샷(Web-5) 로직 무변경.
- 돋보기 마크·순백·Pretendard·코랄. 순위·유사도 무조작·재학습0·키미저장·holdout재개봉0. 왼쪽선0·회색배경0.
- 금지: 엔진·채점기·데이터·결과화면 로직 변경 / 중앙 몰림 / 왼쪽선·회색배경 / 지어낸 기능 / 다른 프로젝트.
