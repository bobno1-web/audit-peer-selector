# LOOP_WEB3 — 결과 화면 그룹핑 (업종 유사 / 규모만 유사)

## 배경·의도
현재 결과는 유사도 점수 순으로만 나열된다. 그래서 규모만 비슷한 엉뚱한 업종
(예: CJ제일제당 → 삼성중공업)이 상위에 섞여 신뢰도가 떨어져 보인다.
개선: peer 를 두 그룹으로 나눠 표시한다.
  - "업종·사업내용이 유사한 기업" 을 위 그룹
  - "규모·시가총액만 유사(업종은 다름)" 를 아래 그룹
"식품회사엔 식품회사가 위에" 보이게 해 납득도를 높인다.

## ★ 절대 원칙 (가장 중요)
이것은 "표시 재구성"이지 "순위 조작"이 아니다.
- 엔진이 낸 유사도 점수·순위 숫자는 절대 바꾸지 않는다(각 peer 의 실제 순위 1~10 과 유사도 그대로).
- 재정렬로 "가짜 순위"를 만들지 않는다. 순위 숫자는 엔진 그대로.
- 그룹 분류는 이미 존재하는 rationale 태그(top_axes)로만 한다. 새 판정 로직·새 점수·새 필드 금지.
- 엔진·채점기·데이터·라우팅·산출물 무변경. 템플릿(화면)만 수정.

## 그룹 분류 규칙 (실제 top_axes 기반 — docs/GROUPING_RULE.md 참조)
- 각 peer 의 `top_axes`(엔진 산출 `rationale` 상위 2축, 양수만)에
  업종(industry) 또는 사업내용(text) 라벨이 있으면 → **섹션 1(업종·사업 유사)**.
- 그 외(규모/시가총액/성장성 위주, 업종·사업내용 없음) → **섹션 2(규모·시총만 유사)**.
- 경계: top_axes 가 비면 업종·사업 신호가 없으므로 섹션 2.
- 판정에 쓰는 라벨명은 결과의 `axis_labels`(industry/text) 를 참조 — 문자열 리터럴 지어냄 0.
