# PREDICTION — 예측 방식 개선 (Loop 6 PART 1, B)

지금까지 5루프는 **"peer 를 어떻게 고르는가"**(선정 축)만 바꿨다. 이번엔 처음으로 **"고른 peer 로
어떻게 예측하는가"**(집계 방식)를 바꿔본다. peer 선정은 **L4 고정**, 예측 집계만 변화시킨다.

- 스크립트: `scripts/loop6_predict.py`. dev(2016~2022) 전용. holdout 미사용.
- ★ 정보 차단벽 유지: 예측은 peer 의 **채점비율 값** + **엔진 유사도 점수**만 쓴다(타겟 비율 미접근).
  유사도 점수는 엔진이 허용 피처로만 계산한 랭킹 점수(engines/similarity `rank(with_sim=True)`) →
  누출 아님. `test_no_leakage` 통과.
- ★ 자기검증: **median 집계 경로가 canonical 채점기(score.py `_cases_from_peers`)와 동일**(APE
  중앙값 0.496465 = 0.496465, |Δ|<1e-9)함을 실증. L4 선정 재현(커밋된 L4 peers 대비 mismatch **0**).
- ★ ORACLE 동결 준수(D-028): ORACLE 은 예측=**median**(평균 아님)을 못박되 **k 는 config(기본 5)**로
  열어둔다. 집계 '함수' 변경은 median 편차, **k 변경은 config 튜닝(가중치처럼 dev 유도)**.

## 변형별 dev APE (peer 선정 = L4 고정)

**집계 함수 (k=5 고정; 교정 targets):**

|---|--:|---|
| (a) median (ORACLE 동결 기본) | **0.4977** | 기준(=canonical 재현) |
| (b) 유사도 가중평균 | 0.5578 | ★ **악화**(가까운 peer에 큰 가중이 오히려 해로움) |
| (c) 유사도 가중중앙값 | 0.4990 | median 과 사실상 동일 |

## ★ 최선 변형 = **median @ k=10**

- **개선의 원천은 오직 k(peer 수)다.** 집계 '함수'를 바꾸는 것(가중평균·가중중앙값·절사)은 **도움이
  안 되거나(≈median) 해롭다(가중평균 +0.06)**. 전조합 argmin 은 weighted_median@k10=0.4781 이지만
  median@k10=0.4794 와 차이 0.001(부트스트랩 잡음 내) → **ORACLE 동결 median 유지, k=10 채택**
  (parsimony + 동결 준수).
- **k↑ 가 median 을 낮추는 이유(정직한 기전):** peer 5→10 개면 중앙값이 **더 안정**(개별 이상 peer의
  영향 완화)해 중심 예측이 정확해진다. coverage 도 0.985→0.998 로 오르지만(유효 peer<3 FAIL 감소),
  FAIL(페널티 2.026)은 상위 꼬리(~1.5%→0.2%)라 **중앙값에는 거의 영향 없다** → k 개선은 coverage
  인공물이 아니라 **중심 예측 정확도 향상**이 본질(SHOWDOWN_L6 비율별 분해로 확인).
- **k 의 U자 상한:** k=3(0.5706, 너무 적어 noisy) → k=10(0.4786, 최적 근처) → k=∞(시장중앙값 0.5720
  로 회귀). 격자 {3,5,7,10} 는 사전등록(config); k=10 이 격자 내 argmin. k=10 에서도 아직 하강 중이라
  더 큰 k 가 조금 더 도울 여지는 있으나, 격자를 결과 보고 늘리지 않는다(튜닝 금지).

## 결론
예측 방식(B)은 **처음으로 상한을 유의하게 밀어낸** 지렛대다. peer 선정(L4)을 그대로 두고 **집계 peer 수만
5→10** 으로 늘려 dev APE 를 **0.4977→0.4794**(교정 targets)로 낮췄다 — L6 vs L4 −3.68%,
firm-clustered 부트스트랩 유의(SHOWDOWN_L6). 단 이는 **집계 함수의 정교화가 아니라 표본 수(k)의 효과** —
"peer-중앙값 예측"의 구조는 그대로다. 남은 격차(→0.433)를 완전히 메우진 못한다(SHOWDOWN_L6 승리조건 참조).
