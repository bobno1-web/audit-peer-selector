# VERIFY — 검증 규칙

검증방(verifier)이 각 루프의 산출물을 판정할 때 따르는 규칙.

## 규칙
1. **채점 기준을 발명하지 않는다.** 개발 지시서에 명시된 "완료 조건"만 채점한다.
   지시서에 없는 항목으로 FAIL을 내면 그것은 검증방의 오류다.
2. **고치지 않는다. 판정만 한다.** 검증방은 코드를 수정하지 않는다.
3. **지시서와 완료조건이 서로 모순되면**, 코드를 FAIL 내지 말고 "지시서 결함"으로 보고한다.
   (근거: Loop 0-A에서 지시서와 검증표 불일치로 오판 FAIL이 발생했다.)
4. **출력 형식**: 항목별 통과/실패 표 + 마지막 줄에 `PASS` 또는 `FAIL`.
5. **채점 기준은 `docs/loops/LOOP_XX.md` 파일이다.** 그 파일이 없으면 채점하지 않고
   **"지시서 부재"로 반려**한다. 채팅·기억은 근거가 아니다.
   (근거: Loop 0-A는 지시서-검증표 불일치, Loop 0-C는 지시서 파일 부재 —
   같은 원인의 사고가 2회 발생했다.)
