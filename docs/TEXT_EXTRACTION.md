# TEXT_EXTRACTION — '사업의 내용' 섹션 정밀 추출 (Loop 3 PART 1)

dev(2016~2022)만. **holdout 미개봉.** 원천: 각 기업의 가장 이른 dev A001(사업보고서) 원문
(rcept_dt ≤ 채점되는 어떤 T → 룩어헤드 0). 파서: `scripts/section_parser.py`.

## 추출 방법 (구조적 — 특정 회사 하드코딩 0)
DART 사업보고서는 규제 서식(자본시장법)에 따라 최상위 섹션을 **로마숫자 제목**으로 나눈다:
`I. 회사의 개요 / II. 사업의 내용 / III. 재무에 관한 사항 / … / XI.`.
- 최상위 섹션 = 로마숫자 접두를 가진 `<TITLE>` (서식의 번호체계 = 구조 앵커).
- 그 중 제목이 표준 섹션명(config `text_section_title: "사업의 내용"`)을 포함하는 섹션을,
  **다음 최상위 섹션 직전까지** 잘라낸다.
- 섹션명·로마숫자는 규제 서식의 라벨이지 **회사명·종목코드가 아니다**(no-hardcoding 준수).
- ★ 추출 실패는 **결측**(빈 문자열 + status) — 임의 대체 0.

## 품질 육안검사 (20 표본, seed=20260715)
- **18~19/20 = 진짜 사업 설명** — 산업의 특성·제품·사업부문·업계 현황이 담겼다
  (예: 백금T&A "초단파수신기…단말기 개발·제조", 인선이엔티 "건설폐기물 재활용", 대우건설 "토목·건축…").
- 잡음은 경미: 일부 원문의 미렌더 DSL 플레이스홀더(`◆click◆…dsl`)·용어설명 표가 섞이나
  **문서 자체의 내용**이며 사업모델 신호를 가리지 않는다.
- SPAC(기업인수목적회사)는 "합병이 유일한 사업목적"만 나온다 — 이는 **정확한 반영**(실사업 없음),
  오류가 아니다. → 추출이 현실을 왜곡 없이 옮긴다.

## Loop 2(전문)와의 차이 — ★ 한 변수만 바꿨다
| | Loop 2 (전문) | Loop 3 (섹션) |
|---|---|---|
| 텍스트 소스 | 사업보고서 **전문의 첫 4,000자** | **"사업의 내용" 섹션만** |
| 실제 내용 | 표지+목차+회사개요(사업설명 도달 전 절단) | 산업·제품·사업부문 서술 |
| 중앙 길이 | 4,000자(상한에 절단, 2837/2841행) | 20,765자 |
| 벡터라이저 | 문자 3-gram TF-IDF, vocab 3000, L2정규화 | **동일**(격리 위해 불변) |

→ 벡터라이저를 **동일하게 고정**하고 **텍스트 소스만** 바꿨다. 결판에서 텍스트 가중치가
  0을 벗어나면, 원인은 (모델이 아니라) **소스 정밀화**임이 확정된다.

## 임베딩·캐시 (api-budget 준수)
