# 동종업계 탐색기

기업 하나를 입력하면 비교가능한 동종기업(peer)을 순위대로 반환하는 엔진과, **그 엔진이 얼마나 잘하는지
자동으로 채점하는 하네스**. 용도는 초도감사(신규 수임)의 분석적 절차 — 감사인이 타겟의 재무비율을
동종기업 대비 이상치로 볼 수 있게 "정상 범위"를 정확히 만든다. 정답 라벨이 없는 문제이므로, peer 의
재무비율 중앙값을 타겟 예측치로 삼아 실제와의 오차(APE)로 peer 품질을 측정한다. 엔진은 갈아치울 수
있고, **채점기는 남는다.**

## 무엇을 만들었나
- **엔진(baseline → L6):** 산업·규모에서 시작해 시가총액·사업내용 텍스트·매출 성장률·부문 구성·peer 수(k)를
  더한 다축 유사도 랭커. 정보 차단벽(엔진은 채점 대상 비율 계정을 못 봄)을 폴더 경계·훅으로 물리 강제.
- **채점·검증 하네스:** 시점(PIT) 데이터층(룩어헤드·생존편향 0), 동결 채점기(`ORACLE.md`), 코드로 강제되는
  게이트(`gate.py`), 출처 결속 검증(`test_provenance_integrity`), dev/holdout 분리.
- **산출물 표현 계층:** peer 목록에 신뢰도 등급·선정 근거(축별 기여)·확인필요지점을 붙인 구조화 출력
  (`OUTPUT_FORMAT.md`).

## 무엇을 실증했나
- **out-of-sample 검증(holdout 1회 개봉):** 최종 엔진이 산업+규모 baseline 을 **−10% 유의하게** 이겼고,
  개발 순위가 holdout 에서 완전히 유지됐다(과적합 아님, 실재하는 개선).
- **상한의 실증:** 야심찬 목표선(baseline −20%)은 미달했다. 6개 축·예측방식·신경망 임베딩을 전부 실측해
  도달한 벽이 median APE ≈**0.50** 이다. 이는 실패가 아니라 "peer 중앙값으로 재무비율 예측"의 **구조적
  상한**의 좌표다 — 남은 오차는 동종그룹 내 사업 이질성이라 peer 선정 정교화로는 못 줄인다.
- **결론:** "더 나은 peer 선정기 + 이상 탐지기"로는 실용 가능(검증됨), 정밀 예측기로는 한계 분명.

## 문서
- **`docs/JOURNEY.md`** — 설계 결정과 판단의 기록(왜 이렇게 만들었나, 하네스가 무엇을 잡았나). ★ 먼저 읽을 것.
- `docs/ORACLE.md` — 채점 명세(동결). `docs/FINAL_REPORT.md` — 최종 성능·사용 가이드.
- `docs/DECISIONS.md` — 설계 결정 이력. `docs/LOOP_LOG.md` — 루프별 진행. `docs/OUTPUT_FORMAT.md` — 산출물 스키마.
- `.claude/rules/` — 코드로 강제되는 프로젝트 규칙(필터·룩어헤드·하드코딩·게이트·출처 금지 규칙).

## 구조
```
engines/     peer 선정 엔진(baseline, similarity). ★ scoring/ 을 import하지 않는다.
scoring/     채점기(ORACLE 구현). 엔진 출력 파일(peers.parquet)만 읽는다.
scripts/     데이터 빌드·루프 탐색·정교화·검증 스크립트.
config/      설정·계정 별칭·동결 파일(상수는 데이터에서 유도).
data/pit/    시점 스냅샷(features=엔진 입력 / targets=채점기 전용). 용량상 미커밋(지문 매니페스트로 검증).
runs/        루프별 산출물(scores.json 등). tests/ 검증. docs/ 문서.
```
